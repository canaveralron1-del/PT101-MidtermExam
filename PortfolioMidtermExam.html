First Encounter with Git

Stage 01

The first time I ran git init, I had no idea what I was doing. The terminal looked intimidating — just a blinking cursor and a sea of unfamiliar commands. I treated it like a mystery box and just kept typing, half expecting it to break my computer.

git init git status

Understanding Commits

Stage 02

Learning what a commit really meant changed everything. It wasn't just "saving" — it was creating a checkpoint in time, a snapshot of your work that you could always return to. I started writing meaningful commit messages instead of the classic "asdfg" or "final final v2".

git add . git commit -m git log
3
upload

Pushing to GitHub — The Big Moment

Stage 03

The first time I successfully ran git push origin main and saw my code live on GitHub — I felt like I'd unlocked a superpower. My work was no longer trapped on my laptop. It was out there in the world, visible, shareable, and real.

✓ Branch 'main' set up to track remote branch 'main' from 'origin'.

Surviving the Merge Conflict

Stage 04

Nothing humbles you faster than your first merge conflict. Seeing those dreaded <<<<<<< HEAD markers felt like staring into chaos. But working through it taught me to think carefully about how code changes interact — a skill that made me a far better collaborator.

CONFLICT detected git merge conflict resolved ✓

Where I Am Now & What's Next

Stage 05

Today, Git is second nature to me. Branching, rebasing, pull requests — these are no longer scary words; they're tools in my daily workflow. My GitHub has become my developer diary — a living record of growth, experiments, and projects I'm proud of. And this repo? It's just the beginning.
